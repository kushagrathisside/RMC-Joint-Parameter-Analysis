# Configuration Reference

The primary runtime configuration lives in [`configs/config.yaml`](../configs/config.yaml).

## Supported keys

### `robot`

```yaml
robot:
  name: panda_mj_description
  variant: scene
```

Fields:

- `name`
  Robot description package name passed to `load_robot_description(...)`

- `variant`
  Optional MuJoCo XML variant. For Panda, `scene` loads the grounded scene wrapper instead of the bare arm XML.

If omitted, the runtime defaults to `panda_mj_description`.

### `controller`

```yaml
controller:
  type: pd
  params:
    kp: 150.0
    kd: 20.0
```

Fields:

- `type`
  Supported runtime values: `pd`, `rl`, `nn_pd`

- `params.kp`
  Proportional gain used by `PDController` and `NNCompensatedPD`

- `params.kd`
  Derivative gain used by `PDController` and `NNCompensatedPD`

Notes:

- `params` are ignored by `RLController`
- `RLController` is a direct torque policy, not a gain-predicting policy anymore
- `rl` requires either `results/policy.h5` plus `results/policy_normalization.npz`, or `results/ppo_policy.h5` plus `results/ppo_normalization.npz`
- `nn_pd` requires `results/residual_policy.h5` plus `results/residual_normalization.npz`
- when `nn_pd` loads a trained residual model, it restores the saved training-time `kp` and `kd` values from the normalization artifact

### `trajectory`

```yaml
trajectory:
  type: classic
  params:
    amplitude: 1.0
    frequency: 0.5
```

Fields:

- `type`
  Supported runtime values: `classic`, `neural`

- `params.amplitude`
  Used by `TrajectoryGenerator`

- `params.frequency`
  Used by `TrajectoryGenerator`

Notes:

- `trajectory.params` are ignored by `NeuralTrajectoryGenerator`
- the trajectory target is part of the unified 21D learning state

### `simulation`

```yaml
simulation:
  duration: 20
```

Fields:

- `duration`
  Total simulated seconds for `scripts/main.py`

Notes:

- the simulation loop advances using the MuJoCo model timestep, not wall-clock timing

### `output`

```yaml
output:
  video: results/simulation.mp4
```

Fields:

- `video`
  Optional output path for the MuJoCo render video generated by `scripts/main.py`

Notes:

- this does not change the learning dataset path
- the transition dataset is saved separately to `results/dataset.npz`

## Example configurations

### PD controller with handcrafted trajectory

```yaml
robot:
  name: panda_mj_description
  variant: scene

controller:
  type: pd
  params:
    kp: 150.0
    kd: 20.0

trajectory:
  type: classic
  params:
    amplitude: 1.0
    frequency: 0.5

simulation:
  duration: 20

output:
  video: results/pd_classic.mp4
```

### Residual controller with handcrafted trajectory

```yaml
robot:
  name: panda_mj_description
  variant: scene

controller:
  type: nn_pd
  params:
    kp: 150.0
    kd: 20.0

trajectory:
  type: classic
  params:
    amplitude: 1.0
    frequency: 0.5

simulation:
  duration: 20
```

### Direct torque RL controller with neural trajectory

```yaml
robot:
  name: panda_mj_description
  variant: scene

controller:
  type: rl

trajectory:
  type: neural

simulation:
  duration: 20
```

## Learning artifact paths

These are fixed in code, not in config:

- dataset: `results/dataset.npz`
- behavior cloning model: `results/policy.h5`
- behavior cloning normalization: `results/policy_normalization.npz`
- behavior cloning metrics: `results/bc_metrics.json`
- residual model: `results/residual_policy.h5`
- residual normalization: `results/residual_normalization.npz`
- residual metrics: `results/residual_metrics.json`
- PPO-style actor: `results/ppo_policy.h5`
- PPO-style normalization: `results/ppo_normalization.npz`
- PPO-style metrics: `results/ppo_metrics.json`
- evaluation summary: `results/evaluation_summary.json`

## Configuration caveats

- changing `controller.type` to anything other than `pd`, `rl`, or `nn_pd` raises `ValueError`
- changing `trajectory.type` to anything other than `classic` or `neural` raises `ValueError`
- selecting `rl` or `nn_pd` without their trained artifacts raises `FileNotFoundError`
- `scripts/generate_video.py` is still tied to the older sample-log path and is not controlled by this config
